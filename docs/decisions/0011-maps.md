# ADR 0011 — Maps: static SVG of Paddington Basin

**Date:** 2026-05-17
**Status:** Accepted

## Decision

The Locations page uses a **hand-crafted SVG** showing Paddington Basin with the two pitches marked as brand-coloured dots. No interactive map tile service in v1.

## Why

- Three pitches, two locations, all within ~250 m. The information density does not justify an interactive map.
- Hand-crafted SVG keeps Lighthouse perf trivially clean (no third-party JS, no tile fetches).
- Adds visual personality vs a generic tile-map embed — the canal becomes a real design element.
- No paid Mapbox / Google Maps key needed. No free-tier-with-limits friction.

## Alternatives rejected

- **MapLibre GL + Stadia Maps tiles** — heavier JS bundle; another network dependency; previous build's procedural-trucks-on-MapLibre direction was rejected for unrelated reasons but this carries some of that feel.
- **Static rasterised map render from Mapbox Static API** — needs an API key (free tier exists but tied to a token in the repo); SVG is simpler.
- **Embed a Google Maps iframe** — privacy concerns, brand-chrome leak.

## Re-evaluation trigger

If the owner adds pitches outside Paddington Basin (e.g. a King's Cross rotation), revisit. Multi-area coverage starts to want an interactive map.
